# 客观化表述替换表

笔记要求**客观第三方视角**，禁用一切主观人称。本表给出完整的替换规则。

## 禁用表述

| 类型 | 示例（禁用） |
|------|------|
| 讲者姓名 | "X 教授指出"、"X 老师强调" |
| 主观人称 | "他承认"、"他指出"、"他回忆" |
| 个人色彩 | "他读博期间"、"他坦承"、"他直言" |
| 临场指代 | "他现场补充"、"他在交流中提到" |

## 推荐表述

| 场景 | 推荐表述 |
|------|---------|
| 引用论断 | 报告指出 / 援引 / 估计 / 直言 |
| 引用论文 | 报告援引 / 研究援引 |
| 临场补充 | 讲座现场补充 / 讲座中提及 |
| 个人经验 | 据讲座所述 / 据讲座分享 |
| 团队工作 | 团队提出 / 团队发现 / 团队调研 |
| 学术共识 | 学术界认为 / 研究显示 |
| Q&A 回应 | **答**：（替代 **X 教授**：） |
| 引用原话 | 用引号包裹原话，但不指明说话人 |

## 完整替换映射

| 原主观表述 | 客观化表述 |
|----------|----------|
| X 教授指出 / 援引 / 估计 | 报告指出 / 援引 / 估计 |
| X 教授回忆 / 坦承 / 直言 | 据讲座所述 / 报告坦承 / 报告直言 |
| X 教授进一步推演 / 据此提出 | 报告进一步推演 / 报告据此提出 |
| 他读博期间曾... | 讲座中提及...的现象 |
| 他承认 / 他提出 | 后来承认 / 团队提出 |
| X 教授现场补充 | 讲座现场补充 |
| X 教授在交流中提到 | 讲座中亦提到 |
| X 教授团队历时最久 | 团队历时最久 |
| **X 教授**：（Q&A） | **答**：（Q&A） |
| X 教授最后强调 | 报告最后强调 |

## 处理讲者原话引用

**保留原话**（用引号包裹），**但不指明说话人**。

❌ 错误：
> X 教授说："AI 编程现在非常烧钱"

✅ 正确：
> 讲座现场补充："AI 编程现在非常烧钱"

或：

> 报告直言："为什么这些乱七八糟的论文越来越多？"

## 处理个人经历

讲者分享的个人经历（如"读博期间曾..."）可以保留，但要改为客观描述。

❌ 错误：
> **某岗位是 X 教授判断"最可能被重构的岗位"**：他早年项目中曾反复处理类似需求，因此认为该岗位会被 AI 改变...

✅ 正确：
> **某岗位被判断为"最可能被重构的岗位"**：讲座中提及，相关工作流中存在大量反复修改、跨角色确认和低效协作现象，因此具备被 AI 工具重构的可能。

## 自检方法

完成笔记后，用以下命令扫描剩余主观表述：

```bash
grep -nE "X 教授|讲者|他(读博|现场|承认|估计|提出|指出|直言|用一个|进一步|据此|回忆|早期|坦承|援引|在切入|判断|强调|最后)" 笔记.md
```

应返回 0 行。如果有残留，按本表替换。
