# 核查清单（reference）

> 阶段 6（多轮核查）使用。三张清单 + 常见坑速查。核查基准层级：`讲稿原文 > 已有单篇笔记 > 摘要`。

---

## A. 内容核查清单（逐源逐条）

对每个源卡片与每条跨源主线，逐项核对：

- [ ] **数据**：每个数字/百分比/规模与源原文一致？（常见错：抄错位、小数点、单位）
- [ ] **归属**：每个"某某提出/认为"确实是该源讲的？（常见错：张冠李戴、把 A 的观点挂到 B）
- [ ] **论文名/venue**：论文标题、作者、会议/期刊、年份与源一致？（常见错：venue 记错，如 ISSTA vs USENIX）
- [ ] **概念定义**：术语定义忠于源？（常见错：用流行解释替代源原话）
- [ ] **图注与图内容**：图注描述与图所在页/内容相符？（常见错：图注描述的是相邻页的内容，页码错位）
- [ ] **跨源连线**：说"A 的 X = B 的 Y"时，两者真的一致？（常见错：过度归并不同概念）
- [ ] **争议/共识**：归纳的"分歧/共识"每个相关源原文都支持？（不能凭印象）

**方法**：派 agent 逐源对照；agent 标"无据"的条目，**回讲稿原文 Grep 复核**（已有笔记是简化版会漏）。

## B. 图表核查清单

### 图片（每张）
- [ ] 路径存在（Bash `test -f` 或脚本验证）
- [ ] 相对路径正确（从笔记文件所在目录算）
- [ ] 图注含页码/位置
- [ ] 图注与正文论述对应（不只装饰）

**⚠️ 坑**：路径含特殊字符（如文件名里的 `(2)`、空格、中文）会让正则提取出错。用脚本批量验证时，对这类路径**单独复核**。

### 锚点（内部链接）
- [ ] 每个 `](#锚点)` 都对应一个实际标题
- [ ] 目录链接 ↔ 标题匹配（GitHub 锚点规则：小写、去标点、空格转 `-`）
- [ ] 改过标题后，同步更新所有引用该锚点处

**方法**：`Grep`](#[^)]+)\)` 提取所有锚点，逐一核对标题存在。

### mermaid 语法
- [ ] 每个 ```mermaid 块语法完整（`graph`/`subgraph`/`classDef`/`class` 配对）
- [ ] `class` 引用的节点 ID 都存在
- [ ] 节点文本特殊字符在 `[""]` 内
- [ ] 改动后（加节点/加层）同步更新 `class` 语句

## C. 连带一致性清单（重构/改动后必做）

> 任何结构性改动（如把某源纳入框架、改章节标题、改计数）后，全文 Grep 检查残留：

- [ ] **计数**：源数量表述一致（如"10 场→11 场"无残留）
- [ ] **归属**：同一源在各处归属一致（如某源在五层、主线、卡片、索引的定位对齐）
- [ ] **术语**：同一概念全文统一译名/写法
- [ ] **跨节引用**：主线/争议里引用的数据，与源卡片里的数据一致
- [ ] **索引同步**：9.x 索引反映实际引用的图/论文（新增图要补进索引）

**方法**：用 `Grep` 扫关键词（源名、计数词、关键数据），看各处是否一致。

## D. 常见坑速查（都是实战教训）

| 坑 | 表现 | 对策 |
|---|---|---|
| 命名/异步 agent 不返回 | 只回 idle notification，无内容 | 用同步 agent（不带 name） |
| 只看单篇笔记误报"无据" | agent 标某数据源里没有 | 回讲稿原文 Grep（笔记是简化版） |
| 跨源归纳张冠李戴 | 把 A 的立场挂到 B | 归纳争议必须回每个源原文确认 |
| 路径含 `(2)` 提取错 | 脚本报"缺失: 2" | 单独 `test -f` 复核该路径 |
| 框架排除某源 | 某源"放不进"分层 | 改框架，不是源的错 |
| 过度归并 | 多个不同概念被等同 | 加注差异，或拆开 |
| 改标题断锚点 | 目录链接指向旧标题 | 改标题同步改锚点引用 |
| 重构后计数残留 | 正文还写旧的"10 场" | Grep 全文扫计数词 |
| 图注页码错位 | 图是 N 页，注写 N±1 | 核对单源素材的图描述页码 |

## 核查报告输出格式

```markdown
# 第 N 轮核查报告

## 检查范围
（哪些源/部分）

## 发现问题
- **[位置]** 笔记原文："..." → 源材料："..." ｜ 问题类型（数据错误/归属错误/...）

## 处理
（每条：修正/保留+依据）

## 核对无误
✓ <部分名> 无误
```

**原则**：每条问题精确引用原文；保留某条时必须给依据（如"讲稿第 X 行有据"）。
